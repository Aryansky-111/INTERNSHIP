Cybersecurity Internship Task 3: Vulnerability Assessment Report

Introduction

This report details a vulnerability assessment performed as part of a cybersecurity internship task. The objective was to identify common vulnerabilities on a system using free tools and to understand the process of vulnerability scanning, risk assessment, and reporting. A scan was conducted using Nmap on the local Linux environment, supplemented by research into common vulnerabilities and the Common Vulnerability Scoring System (CVSS).

Vulnerability Scan (Nmap)

A basic vulnerability scan was performed using Nmap (Network Mapper), a free and open-source utility for network discovery and security auditing. While it is not a full-fledged vulnerability scanner like OpenVAS or Nessus, it can identify open ports and services, which are often indicators of potential vulnerabilities.

Scan Details

  • Tool Used: Nmap
  • Target: Localhost (127.0.0.1)
  • Command: nmap 127.0.0.1

Scan Results

Analysis of Scan Results

The scan identified three open ports:

  • 22/tcp (SSH): Secure Shell (SSH) is a cryptographic network protocol for operating network services securely over an unsecured network. Misconfigurations or weak credentials can lead to vulnerabilities.
  • 5900/tcp (VNC): Virtual Network Computing (VNC) is a graphical desktop sharing system. If VNC is exposed to the internet without proper authentication or encryption, it can pose a security risk.
  • 5901/tcp (VNC-1): Likely another VNC instance. Same security considerations apply.

These open ports indicate services running on the system that could be exploited if not properly secured. Further investigation would be required to determine the specific versions of these services and if any known vulnerabilities exist. Strong authentication and network segmentation are crucial.

Common Computer Vulnerabilities

1. Software Vulnerabilities

  • Outdated or Unpatched Software: Targeted using publicly available exploits.
  • Buffer Overflows: Overwriting adjacent memory, leading to crashes or arbitrary code execution.
  • SQL Injection: Allows viewing or modifying unauthorized data in databases.
  • Cross-Site Scripting (XSS): Injection of malicious scripts into trusted websites.

2. Network Vulnerabilities

  • Misconfigured Firewalls/Routers: Leave ports open or expose internal services.
  • Weak Encryption: Outdated encryption makes data interception easier.
  • Open Ports and Services: Unsecured services are entry points.

3. Human/Process Vulnerabilities

  • Phishing: Tricking users into revealing sensitive data.
  • Weak Passwords: Easy to guess or crack.
  • Lack of Security Awareness: Users unintentionally creating risks.

4. Malware

  • Viruses: Attach to programs and spread on execution.
  • Worms: Spread across networks without interaction.
  • Ransomware: Encrypts files and demands payment.
  • Trojans: Disguised as legitimate software.

Common Vulnerability Scoring System (CVSS)

CVSS is a standard for assessing vulnerability severity. It provides numerical scores (0.0 to 10.0) and qualitative categories.

CVSS Metrics

  1. Base Metrics: Characteristics of the vulnerability (e.g., Attack Vector, Privileges Required).
  2. Temporal Metrics: Change over time (e.g., Exploit Code Maturity, Report Confidence).
  3. Environmental Metrics: Tailored to specific user environments.

CVSS Scoring

  • None: 0.0
  • Low: 0.1 - 3.9
  • Medium: 4.0 - 6.9
  • High: 7.0 - 8.9
  • Critical: 9.0 - 10.0

Interview Questions and Answers

1. What is vulnerability scanning?

A process to identify weaknesses in systems using automated tools to prevent attacks.

2. Difference between vulnerability scanning and penetration testing?

  • Vulnerability Scanning: Automated, finds known issues.
  • Penetration Testing: Manual, simulates real attacks for deeper insights.

3. Common vulnerabilities in personal computers?

  • Outdated software, weak passwords, malware, misconfigured firewalls, phishing, insecure Wi-Fi, lack of antivirus.

4. How do scanners detect vulnerabilities?

Using signature databases, behavioral analysis, policy checks, port scans, and credentialed access.

5. What is CVSS?

A scoring system for assessing vulnerability severity, helping prioritize risks.

6. How often should scans be done?

Based on criticality, compliance, and change frequency. Generally monthly or after changes.

7. What is a false positive?

When a scanner wrongly flags a vulnerability. It requires manual verification.

8. How do you prioritize vulnerabilities?

Based on CVSS score, exploitability, impact, asset importance, and threat intelligence.

Conclusion

This assessment provided a basic understanding of how to identify and prioritize vulnerabilities using tools like Nmap and the CVSS framework. Understanding vulnerabilities and how to assess them is key to maintaining system security.
